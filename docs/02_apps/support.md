# Support 앱 가이드 (support.md)

------------------------------------------------------------------------

# 1. Support 앱 개요

Support 앱은 django_ma 시스템의 **영업지원 기능을 담당하는 앱**이다.

현재 1차 핵심 기능은 **업계정보 페이지**이며, 로그인 직후 모든 사용자가
가장 먼저 접속하는 **공통 랜딩 페이지** 역할을 한다.

이 페이지는 보험 업계 관련 뉴스 정보를 수집·정리하여 제공하고, 사용자
선호도를 기반으로 **개인화 추천 기능**을 제공한다.

즉 Support 앱은 단순 뉴스 게시판이 아니라 다음 기능을 수행한다.

-   보험 업계 뉴스 자동 수집
-   기사 정리 및 분류
-   사용자 선호도 수집
-   개인화 기사 추천
-   업계 정보 탐색 허브 제공

------------------------------------------------------------------------

# 2. 시스템 내 위치

Support 앱은 다음과 같은 시스템 위치를 가진다.

  역할               설명
  ------------------ ----------------------------------------
  최초 랜딩페이지    로그인 후 모든 사용자가 가장 먼저 접속
  정보 허브          보험 업계 뉴스 및 동향 제공
  행동 데이터 수집   사용자 선호도 및 행동 로그 수집
  추천 엔진 입력     추천 알고리즘 데이터 공급

------------------------------------------------------------------------

# 3. 최초 랜딩페이지 정책

Support 앱의 **업계정보 페이지는 시스템 최초 진입 페이지**이다.

### 로그인 흐름

    login 성공
          ↓
    support:industry_info

### 접근 권한

  권한          접근
  ------------- ------
  superuser     가능
  main_admin    가능
  sub_admin     가능
  head          가능
  leader        가능
  일반 사용자   가능

즉 **모든 로그인 사용자가 조회 가능한 페이지**이다.

# 4. 페이지 구성

업계정보 페이지는 다음 UI 구조를 가진다.

## 4.1 추천 기사 영역

사용자 선호도 기반 추천 기사 노출

표시 정보

-   제목
-   언론사
-   요약

    최근 높게 평가하신 자동차보험 관련 기사와 유사합니다

추천 데이터가 없는 경우 fallback 적용

    최근 인기 기사

# 6. 뉴스 수집 구조

뉴스 데이터는 **사용자 요청 시 API 호출하지 않는다**

수집 구조

    Naver News API
          ↓
    Celery 수집 작업
          ↓
    SupportArticle 저장
          ↓
    업계정보 페이지 조회

즉 **배치 수집 + DB 조회 구조**이다.

# 9. 추천 시스템

추천은 다음 단계로 구성된다.

### 1차 추천 (MVP)

규칙 기반 추천

요소

-   최근 기사 가중치
-   사용자 토픽 선호
-   언론사 선호
-   콘텐츠 유사도
-   숨김 기사 제외

------------------------------------------------------------------------

### 추천 점수 예시

    추천점수 =
    최근성 점수 +
    토픽 선호 점수 +
    콘텐츠 유사도 +
    언론사 선호 -
    숨김 패널티

------------------------------------------------------------------------

### 추천 버전 관리

추천 로직 변경 시

    model_version

값을 변경한다.

예시

    rule_v1
    rule_content_v2
    hybrid_v3
