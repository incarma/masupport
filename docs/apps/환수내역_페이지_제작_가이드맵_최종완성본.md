# 환수내역 페이지 제작 가이드맵 — 최종완성본

> **앱**: `commission`  
> **페이지 URL**: `/commission/collect/notice/`  
> **URL name**: `commission:collect_notice`  
> **권한**: `superuser` 전용  
> **처리 방식**: 클라이언트 전용 처리  
> **핵심 라이브러리**: SheetJS `vendor/xlsx/0.18.5/xlsx.full.min.js`  
> **최종 반영 기준**: 대상자 카드 기준 연월 선택, 공통 대상자 검색 모달 연동, 행 추가/파일 선택, 결과 xlsx 생성, 마스킹/금액/지급율 포맷, JS 초기화 순서 버그 대응

---

## 0. 목적과 최종 동작 정의

`commission:collect_notice` 페이지는 환수 원장 파일을 브라우저에서 직접 읽어, 특정 대상자에게 전달할 수 있는 **환수내역 안내자료 xlsx 파일**을 생성하는 페이지입니다.

이 페이지는 서버에 파일을 업로드하지 않습니다. 사용자가 브라우저에서 파일을 선택하면 SheetJS가 로컬 메모리에서 파일을 파싱하고, 결과 xlsx를 Blob으로 생성한 뒤 별도 다운로드 버튼을 통해 저장합니다.

### 최종 사용자 흐름

1. `/commission/collect/notice/` 접속
2. 대상자 카드에서 **기준 연도/월도** 선택
3. `대상자 검색` 버튼 클릭
4. 공통 대상자 검색 모달에서 대상자 선택
5. 내역 파일 카드에서 `+ 행 추가` 클릭
6. 각 행별 원장 기준 연도/월도 선택
7. 파일 선택
8. `안내자료 제작` 클릭
9. 제작 완료 카드 노출
10. `엑셀 다운로드` 클릭

### 최종 결과 파일 핵심 규칙

| 항목 | 최종 규칙 |
|---|---|
| A1 제목 기준 연월 | 대상자 카드의 `noticeTitleYear`, `noticeTitleMonth` 값 사용 |
| A1 제목 문구 | `{소속} {성명} YYYY년 MM월 기준 환수내역` |
| A1 스타일 | 데이터보다 한 단계 큰 폰트, 굵게 |
| C열 상품명 너비 | Excel 기준 `wch: 50` |
| D열 증권번호 | 마지막 4글자만 노출, 앞부분 `*` 마스킹 |
| E/M/N열 이름 | 1글자: `*`, 2글자: 첫 글자 유지 + `*`, 3글자 이상: 첫/마지막 글자 유지, 중간 `*` |
| I/K열 금액 | 1,000 단위 콤마 적용 |
| J열 지급율 | `%` 기호 삽입 |
| 지급금액 0 또는 빈 값 | 결과에서 제외 |
| 마지막 합계 행 `전체 N건` | 결과에서 제외 |
| 파일 저장 | 자동 다운로드 아님. 제작 후 별도 다운로드 버튼 |

## 2. 백엔드 구현 명세

### 2-1. `commission/urls.py`

`collect/notice/` URL을 신규 추가합니다. 기존 URL name은 변경하지 않습니다.

```python
# commission/urls.py

path("collect/notice/", views.collect_notice, name="collect_notice"),
```

권장 위치는 `collect/` URL 그룹 하단입니다.

```python
# =========================================================================
# Collect Notice (환수내역 안내자료)
# superuser 전용 · 클라이언트 전용 xlsx 생성 · 서버 파일 전송 없음
# =========================================================================
path("collect/notice/", views.collect_notice, name="collect_notice"),
```

### 2-2. `commission/views/pages.py`

```python
from django.utils import timezone
from accounts.decorators import grade_required

@grade_required("superuser")
def collect_notice(request):
    """
    환수내역 안내자료 제작 페이지.

    - superuser 전용
    - 파일은 서버로 업로드하지 않음
    - SheetJS가 브라우저에서 직접 파싱/생성
    - 공통 대상자 검색 모달 연동을 위해 accounts_search_url 주입
    """
    ctx = {
        "accounts_search_url": _accounts_search_url(),
        "current_year": timezone.localdate().year,
    }
    return render(request, "commission/collect_notice.html", ctx)
```

#### 중요

- `@grade_required("superuser")`는 내부에서 로그인 요구를 처리하는 프로젝트 규약을 따릅니다.
- 별도 파일 업로드 처리 뷰를 만들지 않습니다.
- audit 로그는 현 단계에서 남기지 않습니다. 서버 전송이 없기 때문입니다.

### 2-3. `commission/views/__init__.py`

기존 lazy import 구조를 유지하고 `_PAGES`에 항목만 추가합니다.

```python
_PAGES = {
    ...
    "collect_notice": ("commission.views.pages", "collect_notice"),
}
```

절대 일반 import 구조로 바꾸지 않습니다.

### 3-3. 템플릿 주의사항

#### 공통 모달 include 위치

잘못된 위치:

```html
{% endblock %}
{% include 'components/search_user_modal.html' %}
```

올바른 위치:

```html
{% include 'components/search_user_modal.html' with search_url=accounts_search_url %}
{% endblock %}
```

`extends` 템플릿에서 include가 block 밖에 있으면 base 템플릿 구조에 따라 렌더링되지 않거나 의도하지 않은 위치에 들어갈 수 있습니다.

#### 행 템플릿 hidden 처리

`noticeRowTemplate`은 `hidden` 상태로 존재하고, JS가 `cloneNode(true)`로 복제합니다. CSS에서 다음 규칙이 반드시 있어야 합니다.

```css
#collect-notice .notice-row[hidden] {
  display: none !important;
}
```

Bootstrap의 `.d-flex`가 `hidden`을 시각적으로 이길 수 있기 때문입니다.

## 4. 공통 대상자 검색 모달 연동 — `search_user_modal.js`

### 4-1. collect-notice는 row 기반 페이지가 아님

`search_user_modal.js`는 기본적으로 Partner/Board 형태의 `tr.input-row` 자동 채우기를 지원합니다. 그러나 `collect-notice`는 대상자 카드만 갱신하면 됩니다.

따라서 `collect-notice`는 다음 방식으로 처리합니다.

- 검색 버튼 클릭: Bootstrap modal open
- 검색 결과 클릭: `userSelected` CustomEvent dispatch
- row 자동입력: 수행하지 않음
- 모달 닫기
- 검색어/결과 초기화

### 4-2. `getActiveRoot()`에 collect-notice 포함

```javascript
function getActiveRoot() {
  return (
    document.getElementById("manage-structure") ||
    document.getElementById("manage-rate") ||
    document.getElementById("manage-efficiency") ||
    document.getElementById("manage-calculate") ||
    document.getElementById("manage-table") ||
    document.getElementById("collect-home") ||
    document.getElementById("collect-notice") ||
    document.getElementById("deposit-home") ||
    document.getElementById("support-form") ||
    null
  );
}
```

### 4-3. 결과 선택 처리부 collect-notice 예외

`resultsBox.addEventListener("click", ...)` 내부에서 `deposit-home` 또는 일반 row autofill보다 먼저 처리합니다.

```javascript
const root = getActiveRoot();

// collect-notice: row 자동입력 없음 → 이벤트만 전달
if (root?.id === "collect-notice") {
  dispatchUserSelected(selected);
  tryHideModal(modalEl);

  if (input) input.value = "";
  resultsBox.innerHTML = "";
  return;
}
```

### 4-4. target row not found 경고 방지

`collect-notice`는 위에서 `return`되므로 아래 일반 row 자동입력 블록을 타지 않습니다.

```javascript
const row = resolveTargetRow(root);
if (!row) {
  console.warn("[search_user_modal] target row not found");
} else {
  autofillSelectedUser(row, selected);
}
```

만약 `collect-notice`에서 `target row not found`가 계속 보이면 다음을 의심합니다.

1. 정적파일 캐시로 이전 `search_user_modal.js`가 로드됨
2. `getActiveRoot()`에 `collect-notice`가 누락됨
3. `collect-notice` 예외 처리 위치가 일반 row 자동입력 이후에 있음
4. 페이지에 `id="collect-notice"` 루트가 없음

## 5. JavaScript 최종 명세 — `static/js/commission/collect_notice.js`

### 5-1. 가장 중요한 버그: `_boot()` 실행 위치

#### 잘못된 코드

```javascript
const root = document.getElementById("collect-notice");

function _boot() {
  ...
  _init();
}

_boot();

const YEAR_RANGE = 3;
const CURRENT_YEAR = parseInt(root.dataset.currentYear, 10) || new Date().getFullYear();
```

이 구조는 다음 오류를 발생시킵니다.

```text
Uncaught ReferenceError: Cannot access 'CURRENT_YEAR' before initialization
```

원인은 JavaScript의 TDZ(Temporal Dead Zone)입니다. `_boot()`가 먼저 실행되어 `_initTitleYmSelects()`가 `CURRENT_YEAR`를 읽는데, 아직 `CURRENT_YEAR`가 선언되기 전입니다.

#### 올바른 코드

```javascript
const root = document.getElementById("collect-notice");
if (!root) throw new Error("[collect_notice] 루트 요소(#collect-notice) 없음");

function _boot() {
  if (root.dataset.inited === "1") return;
  root.dataset.inited = "1";
  _init();
}

window.addEventListener("pageshow", (e) => {
  if (e.persisted) {
    root.dataset.inited = "";
    _boot();
  }
});

const YEAR_RANGE = 3;
const CURRENT_YEAR =
  parseInt(root.dataset.currentYear, 10) || new Date().getFullYear();

let _rowCounter = 0;
let _blobUrl = null;
let _blobName = "";

// 반드시 상수/상태 선언 이후 실행
_boot();
```

### 5-2. 최종 초기화 패턴

```javascript
function _init() {
  _initTitleYmSelects();
  document.addEventListener("userSelected", _onUserSelected);

  root.addEventListener("click", (e) => {
    const btn = e.target?.closest?.("button");
    if (!btn) return;

    if (btn.id === "btnAddRow") {
      _addRow();
    } else if (btn.id === "btnMakeNotice") {
      _onMakeNotice();
    } else if (btn.id === "btnDownloadResult") {
      _onDownload();
    } else if (btn.id === "btnResetDownload") {
      _resetDownload();
    } else if (btn.classList.contains("notice-row-delete")) {
      const row = btn.closest(".notice-row");
      if (row) {
        row.remove();
        _syncEmptyState();
        _resetDownload();
      }
    }
  });
}
```

#### 설계 이유

- 개별 버튼에 직접 `addEventListener`를 여러 개 거는 대신 `root` 위임으로 처리합니다.
- 다운로드 카드가 hidden이어도 이벤트 위임은 안전합니다.
- 동적으로 생성되는 행 삭제 버튼도 별도 바인딩 없이 처리됩니다.
- BFCache 재진입 시 `root.dataset.inited`로 중복 바인딩을 막습니다.

### 5-3. 기준 연월 선택 초기화

```javascript
function _initTitleYmSelects() {
  const yearSel = document.getElementById("noticeTitleYear");
  const monthSel = document.getElementById("noticeTitleMonth");
  if (!yearSel || !monthSel) return;

  yearSel.innerHTML = "";
  for (let y = CURRENT_YEAR + 1; y >= CURRENT_YEAR - YEAR_RANGE; y--) {
    const opt = document.createElement("option");
    opt.value = y;
    opt.textContent = `${y}년`;
    if (y === CURRENT_YEAR) opt.selected = true;
    yearSel.appendChild(opt);
  }

  monthSel.innerHTML = "";
  const defaultMonth = new Date().getMonth() + 1;
  for (let m = 1; m <= 12; m++) {
    const opt = document.createElement("option");
    opt.value = m;
    opt.textContent = `${m}월`;
    if (m === defaultMonth) opt.selected = true;
    monthSel.appendChild(opt);
  }

  yearSel.addEventListener("change", _resetDownload);
  monthSel.addEventListener("change", _resetDownload);
}
```

### 5-4. 대상자 선택 반영

```javascript
function _onUserSelected(e) {
  const { id, name, branch, affiliation_display, affiliationDisplay } = e.detail || {};
  const aff = affiliation_display || affiliationDisplay || branch || "";

  document.getElementById("targetEmpId").value = id || "";
  document.getElementById("targetName").value = name || "";
  document.getElementById("targetBranch").value = aff;

  const displayEl = document.getElementById("targetDisplay");
  displayEl.textContent = aff ? `${name} (${aff})` : name;
  displayEl.classList.remove("text-muted");
  displayEl.classList.add("fw-semibold");

  _resetDownload();
```

```javascript
function _cleanRows(rawRows, ym) {
  const result = [];

  for (let i = 0; i < rawRows.length; i++) {
    const row = rawRows[i];

    // 1행 헤더 제거
    if (i === 0) continue;

    // 마지막 합계 행 제거
    if (/^전체\s*\d+\s*건/.test(String(row[0] ?? "").trim())) continue;

    // 지급금액 0 또는 빈 값 제거
    const pay = row[12];
    if (pay === "" || pay === null || pay === undefined) continue;
    if (typeof pay === "number" && pay === 0) continue;
    if (String(pay).trim() === "0") continue;

    result.push({
      _ym: ym,
      항목구분: _str(row[1]),
      상품명: _str(row[4]),
      증권번호: _maskPolicy(_str(row[5])),
      계약자: _maskName(_str(row[6])),
      수납구분: _str(row[7]),
      영수일: _str(row[8]),
      회차: _str(row[9]),
      영수보험료: _money(row[10]),
      지급율: _rate(row[11]),
      지급금액: _money(row[12]),
      보험계약일: _str(row[13]),
      모집자: _maskName(_stripParenId(_str(row[14]))),
      지급자: _maskName(_stripParenId(_str(row[15]))),
    });
  }

  return result;
}
```

## 7. 마스킹 및 포맷 함수 최종 기준

### 7-1. 문자열 변환

```javascript
function _str(v) {
  if (v === null || v === undefined) return "";
  return String(v).trim();
}
```

### 7-2. 모집자/지급자 괄호 사번 제거

```javascript
function _stripParenId(v) {
  return v.replace(/\([^)]*\)/g, "").trim();
}
```

예시:

| 입력 | 결과 |
|---|---|
| `홍길동(2433672)` | `홍길동` |
| `홍길동 (2433672)` | `홍길동` |

### 7-3. 이름 마스킹

```javascript
function _maskName(name) {
  if (!name) return "";
  if (name.length === 1) return "*";
  if (name.length === 2) return name[0] + "*";
  return name[0] + "*".repeat(name.length - 2) + name[name.length - 1];
}
```

예시:

| 입력 | 출력 |
|---|---|
| `김` | `*` |
| `김민` | `김*` |
| `홍길동` | `홍*동` |
| `강나경희` | `강**희` |

> 주의: 이전 설계안의 “첫 번째, 세 번째 글자 유지” 규칙은 폐기합니다. 최종 기준은 **첫 글자와 마지막 글자 유지**입니다.

### 7-4. 증권번호 마스킹

```javascript
function _maskPolicy(policy) {
  if (!policy) return "";
  if (policy.length <= 4) return "*".repeat(policy.length);
  return "*".repeat(policy.length - 4) + policy.slice(-4);
}
```

예시:

| 입력 | 출력 |
|---|---|
| `123` | `***` |
| `1234` | `****` |
| `1234567890` | `******7890` |

### 7-5. 금액 콤마 포맷

```javascript
function _toNumber(v) {
  if (v === null || v === undefined || v === "") return null;
  if (typeof v === "number") return Number.isFinite(v) ? v : null;

  const n = Number(String(v).replaceAll(",", "").replace("%", "").trim());
  return Number.isFinite(n) ? n : null;
}

function _money(v) {
  const n = _toNumber(v);
  if (n === null) return _str(v);
  return n.toLocaleString("ko-KR");
}
```

예시:

| 입력 | 출력 |
|---|---|
| `1000` | `1,000` |
| `1234567` | `1,234,567` |
| `"1,000"` | `1,000` |

### 7-6. 지급율 `%` 삽입

```javascript
function _rate(v) {
  const s = _str(v);
  if (!s) return "";
  if (s.includes("%")) return s;
  return `${s}%`;
}
```

## 11. 주요 트러블슈팅 기록 및 재발 방지

### 11-1. 기준 연월 드롭다운/행 추가/대상자 반영이 모두 안 되는 경우


#### 원인

`_boot()`가 `CURRENT_YEAR` 선언보다 먼저 실행됨.

#### 해결

`_boot()` 호출을 `CURRENT_YEAR`, `_rowCounter`, `_blobUrl`, `_blobName` 선언 뒤로 이동.

```javascript
const YEAR_RANGE = 3;
const CURRENT_YEAR = parseInt(root.dataset.currentYear, 10) || new Date().getFullYear();

let _rowCounter = 0;
let _blobUrl = null;
let _blobName = "";

_boot();
```

#### 원인 후보

1. `collect_notice.js`가 초기화 오류로 중단됨
2. `search_user_modal.js`에서 `collect-notice` 예외 처리 누락
3. `collect_notice.js`에서 `document.addEventListener("userSelected", _onUserSelected)` 누락
4. 정적파일 캐시

#### 확인 코드


```javascript
if (root?.id === "collect-notice") {
```

#### 설명

#### 해결

## 15. 향후 확장 옵션

### 15-1. audit 로그 확장

현재는 서버 POST가 없어서 audit 로그를 남기지 않습니다.  
추후 감사 추적이 필요하면 “제작 완료” 시점에 아래 최소 정보만 서버로 POST하는 API를 추가할 수 있습니다.

```json
{
  "target_emp_id": "1611851",
  "target_name": "홍길동",
  "base_ym": "2026-05",
  "row_count": 34
}
```

이 경우 `ACTION.COMMISSION_COLLECT_NOTICE_MAKE` 같은 audit 상수를 별도 추가해야 합니다.

### 15-2. ExcelJS 도입

SheetJS Community는 스타일 지원이 제한적입니다. 테두리, 배경색, 정렬 등 완전한 엑셀 서식이 필요하면 ExcelJS 도입을 검토합니다. 다만 기존 검토에서 CSP `unsafe-eval` 이슈가 있었으므로 현재 최종 기준은 SheetJS 단독 유지입니다.

1. `collect_notice.js`의 `_boot()` 위치를 상수 선언 위로 올리면 안 됩니다.

6. 서버 업로드 API를 만들지 않습니다.
7. 기존 `collect_home`과 결합하지 않습니다.
