## 1. 요청 분류 및 전제 확인

**분류**: 기능 확장 설계 (D타입 — 신규 페이지 제작)

| 항목 | 결정값 |
|---|---|
| URL 경로 | `/commission/collect/notice/` |
| URL name | `commission:collect_notice` |
| 접근 권한 | `superuser`만 접근 가능 |
| 권한 데코레이터 | `@login_required` + `@grade_required("superuser")` |
| 페이지 구조 | 기존 `collect_home`과 완전히 별개인 신규 독립 페이지 |
| 템플릿 파일 | `commission/collect_notice.html` |
| 레이아웃 블록 | `{% block content %}` (표준 `container`, 와이드 레이아웃 불필요) |
| 엑셀 처리 방식 | **클라이언트 전용** — 브라우저에서 SheetJS로 파싱, xlsx 생성 후 별도 다운로드 버튼으로 저장 |
| 서버 파일 전송 | **없음** — 업로드된 파일이 서버로 전달되지 않음 |
| CSS 스코프 | `#collect-notice` 루트 ID 하위, `static/css/apps/commission.css`에 추가 |
| JS 패턴 | ESM (`type="module"`) · BFCache 가드 필수 |
| SheetJS 벤더 | 기존 `vendor/xlsx/0.18.5/xlsx.full.min.js` 재사용 |
| 감사 로그 | 서버 전송 없음으로 인해 audit 로그 미적용 (추후 확장 가능) |

### 다운로드 방식 변경 확인 (수정 요건 반영)

기존 `XLSX.writeFile()` 방식(즉시 자동 다운로드)에서 **별도 다운로드 버튼** 방식으로 변경합니다.

| 기존 방식 (자동 다운로드) | 변경 방식 (버튼 클릭 다운로드) |
|---|---|
| '안내자료 제작' 클릭 즉시 파일 저장 | '안내자료 제작' 클릭 → 결과물을 메모리(Blob URL)에 보관 |
| 파일 확인 없이 저장됨 | 별도 '엑셀 다운로드' 버튼이 활성화되어 사용자가 명시적으로 저장 |
| 제작 후 재다운로드 불가 | 페이지 이탈 전까지 반복 다운로드 가능 |

## 2. Raw 파일 구조 분석

> 첨부된 `명세내역_20260504082017_1611851.xlsx` 파일을 직접 파싱하여 확인한 실제 구조입니다.
> JS에서 `XLSX.utils.sheet_to_json(ws, { header: 1 })` 로 파싱 시 **0-indexed 배열**로 접근합니다.

### 2-1. 시트 구조

| 항목 | 값 |
|---|---|
| 시트명 | `Sheet1` |
| 총 행 수 | 36행 (1행 헤더 + 34행 데이터 + 1행 합계) |
| 총 열 수 | 20열 |

### 2-2. 헤더 행 (1행) 전체 열 구조

| Excel 열 | 0-indexed | 헤더명 | 결과물 사용 여부 |
|---|---|---|---|
| A | 0 | 업무구분 | ✗ |
| B | 1 | 항목구분 | ✅ → 결과 B열 |
| C | 2 | 지급/환수 | ✗ |
| D | 3 | 보험사 | ✗ |
| E | 4 | 상품명/차번호 | ✅ → 결과 C열 |
| F | 5 | 증권번호 | ✅ → 결과 D열 (마스킹) |
| G | 6 | 계약자 | ✅ → 결과 E열 (마스킹) |
| H | 7 | 수납구분 | ✅ → 결과 F열 |
| I | 8 | 영수일 | ✅ → 결과 G열 |
| J | 9 | 회차 | ✅ → 결과 H열 |
| K | 10 | 영수보험료 | ✅ → 결과 I열 |
| L | 11 | 지급율(환수율) | ✅ → 결과 J열 |
| **M** | **12** | **지급금액(환수금액)** | ✅ → 결과 K열 / **0 필터링 기준** |
| N | 13 | 보험계약일 | ✅ → 결과 L열 |
| O | 14 | 모집자 | ✅ → 결과 M열 (괄호+사번 제거, 이름 마스킹) |
| P | 15 | 지급자 | ✅ → 결과 N열 (괄호+사번 제거, 이름 마스킹) |
| Q | 16 | 수정률 | ✗ |
| R | 17 | 납기 | ✗ |
| S | 18 | 만기 | ✗ |
| T | 19 | 부서명 | ✗ |

### 2-3. 특이 구조 확인 사항

- **마지막 행**: A열에 `전체 34 건` 형식 합계 문구 → 결과 파일 생성 시 **제거 필수**
  - 탐지 패턴: `String(row[0]).startsWith("전체")` 또는 정규식 `/^전체\s*\d+\s*건/`
- **지급금액(환수금액) = 0 행**: M열(index 12) 값이 `0`이거나 빈 값인 행 → **결과 파일에서 제거**
- **모집자/지급자 형식**: `홍길동(2433672)` → `_stripParenId()` 함수로 괄호 및 사번 제거 후 이름만 추출
- **데이터 시작 행**: 1행이 헤더이므로 데이터는 index 1번(2행)부터 시작

## 4. 백엔드 구현 명세

### 4-1. URL 등록 (`commission/urls.py`)

기존 `urlpatterns` 리스트에 아래 1줄을 추가합니다.

```python
# commission/urls.py — urlpatterns 내 추가
path("collect/notice/", views.collect_notice, name="collect_notice"),
```

**회귀 위험 검토**: 기존 26개 URL과 prefix 충돌 없음. `collect/` 하위 신규 경로로 기존 패턴에 영향 없음.

### 4-2. 뷰 함수 추가 (`commission/views/pages.py`)

```python
# commission/views/pages.py 내 추가
from datetime import date

@login_required
@grade_required("superuser")
def collect_notice(request):
    """
    환수내역 안내자료 제작 페이지.
    - superuser 전용
    - 파일 처리는 클라이언트(SheetJS)에서 전담 — 서버 업로드 없음
    - 대상자 검색 모달 연동을 위해 accounts_search_url context 주입
    """
    from accounts.search_api import get_search_user_url
    context = {
        "accounts_search_url": get_search_user_url(request),
        "current_year": date.today().year,
    }
    return render(request, "commission/collect_notice.html", context)
```

**설계 원칙**:
- 비즈니스 로직 없음 (전량 클라이언트 처리)
- `accounts_search_url`은 `collect_home`과 동일 패턴 재사용
- `current_year`는 기준 연월 셀렉트 초기값 세팅에 사용

### 4-3. `__init__.py` lazy import 추가

`commission/views/__init__.py`의 `_VIEW_MAP` 딕셔너리에 아래 1쌍만 추가합니다.

```python
# commission/views/__init__.py — _VIEW_MAP 딕셔너리 내 추가
"collect_notice": ("commission.views.pages", "collect_notice"),
```

> ⚠️ **절대 준수**: `__init__.py`의 `__getattr__` 기반 lazy import 구조는 순환 참조 방지 및 501 stub 안전망을 위한 의도적 설계입니다. **일반 `import` 문으로 변경하지 않으며**, 기존 딕셔너리 구조 그대로 키-값 1쌍만 추가합니다. 이 원칙을 위반하면 Django 시작 시 순환 참조 오류 발생 위험이 있습니다.

```javascript
// collect_notice.js 내 userSelected 이벤트 수신
document.addEventListener("userSelected", (e) => {
  const { id, name, branch } = e.detail;
  document.getElementById("targetUserId").value   = id;
  document.getElementById("targetUserName").value = name;
  document.getElementById("targetBranch").value   = branch;
  document.getElementById("targetDisplay").innerHTML =
    `<strong>${_esc(name)}</strong> `
    + `<span class="text-muted">(${_esc(branch)})</span>`;
  // 대상자 변경 시 기존 제작된 결과물 초기화
  resetDownload();
});
```

### 6-4. `storeResult()` 및 `downloadResult()` 구현

```javascript
// 메모리 저장 (모듈 스코프 변수)
let _resultBlobUrl = null;
let _resultFilename = null;

function storeResult(blob, filename) {
  // 기존 Blob URL이 있으면 해제 (메모리 누수 방지)
  if (_resultBlobUrl) URL.revokeObjectURL(_resultBlobUrl);

  _resultBlobUrl = URL.createObjectURL(blob);
  _resultFilename = filename;

  // 다운로드 카드 표시
  document.getElementById("noticeDownloadCard").hidden = false;
  document.getElementById("noticeDownloadFilename").textContent = filename;
}

function downloadResult() {
  if (!_resultBlobUrl || !_resultFilename) return;

  // 임시 <a> 태그로 다운로드 트리거
  const a = document.createElement("a");
  a.href = _resultBlobUrl;
  a.download = _resultFilename;
  document.body.appendChild(a);
  a.click();
  document.body.removeChild(a);
}

function resetDownload() {
  if (_resultBlobUrl) {
    URL.revokeObjectURL(_resultBlobUrl);
    _resultBlobUrl = null;
  }
  _resultFilename = null;
  document.getElementById("noticeDownloadCard").hidden = true;
  document.getElementById("noticeDownloadFilename").textContent = "";
}
```

### 6-5. 마스킹 함수 전체 명세

```javascript
/**
 * 증권번호 마스킹
 * 마지막 4글자 제외한 나머지를 '*'로 치환
 * 예) "302609644" → "*****9644"
 */
function _maskId(str) {
  if (!str) return str;
  const s = String(str);
  if (s.length <= 4) return s;
  return "*".repeat(s.length - 4) + s.slice(-4);
}

/**
 * 이름 마스킹
 * - 3자 이상: 첫 번째, 세 번째 글자 유지 / 나머지 '*'
 *   예) "홍길동" → "홍*동", "박명근" → "박*근", "강나경희" → "강*경*"
 * - 2자: 두 번째 글자 '*'
 *   예) "김민" → "김*"
 * - 1자: 그대로 반환
 */
function _maskName(str) {
  if (!str) return str;
  const s = String(str).trim();
  if (s.length === 1) return s;
  if (s.length === 2) return s[0] + "*";
  // 3자 이상: index 0, 2 유지 / 나머지 '*'
  return s.split("").map((c, i) => (i === 0 || i === 2) ? c : "*").join("");
}

/**
 * 괄호 및 괄호 내 텍스트(사번) 제거
 * 예) "박명근(2433672)" → "박명근"
 * 예) "홍길동 (1234567)" → "홍길동"
 */
function _stripParenId(str) {
  if (!str) return str;
  return String(str).replace(/\s*\([^)]*\)/g, "").trim();
}

/**
 * XSS 방어용 HTML escape
 */
function _esc(str) {
  if (!str) return "";
  return String(str)
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;");
}

/**
 * 숫자 2자리 zero-padding
 * 예) 5 → "05", 12 → "12"
 */
function _pad2(n) {
  return String(n).padStart(2, "0");
}
```

## 9. 정정 사항 및 설계 제안

### 9-1. ⚠️ 요구조건 K열 매핑 오류 정정 (확인 필수)

**원문 요구조건 항목 11번**:

> `K열(K4~)에는 '지급금액(환수금액)' 데이터 삽입 (raw 파일 기준 **L열** 데이터)`

**실제 raw 파일 구조 확인 결과**:

| 열 | 헤더명 | 0-indexed |
|---|---|---|
| L열 (index 11) | 지급율(환수율) | 11 |
| **M열 (index 12)** | **지급금액(환수금액)** | **12** |

**정정 내용**: `지급금액(환수금액)` 데이터는 raw 파일의 **M열(0-indexed: 12)** 에 있습니다. L열(index 11)은 `지급율(환수율)` 입니다. 이 가이드맵에서는 아래와 같이 정정하여 반영합니다.

| 결과 열 | 헤더 | raw 0-indexed | 비고 |
|---|---|---|---|
| J | 지급율(환수율) | 11 (L열) | 원문 항목 10번 |
| **K** | **지급금액(환수금액)** | **12 (M열)** | **원문 항목 11번 정정** |

> 구현 착수 전 원문 요구조건과 실제 raw 파일을 직접 대조하여 최종 확인하시기 바랍니다.

| 폰트 크기(sz) | ⚠️ 부분 지원 | ✅ 완전 지원 |
| 셀 테두리(border) | ❌ 커뮤니티 버전 미지원 | ✅ 완전 지원 |
| 셀 배경색 | ❌ 커뮤니티 버전 미지원 | ✅ 완전 지원 |
| 헤더 배경색 | ❌ 커뮤니티 버전 미지원 | ✅ 완전 지원 |

요구조건의 "헤더 행 포함 데이터테이블 전체 테두리 효과 적용"과 "A1 셀 14pt 굵게"를 완전히 구현하려면 SheetJS만으로는 한계가 있습니다.

#### 제안: ExcelJS CDN 추가 (이 페이지 전용)

```html
<!-- collect_notice.html extra_js 블록 내 -->
<!-- ExcelJS: SheetJS 대신 사용하거나, 스타일 적용 전용으로 병행 사용 -->
<script src="https://cdnjs.cloudflare.com/ajax/libs/exceljs/4.3.0/exceljs.min.js"></script>
```

**채택 근거**:
- `cdnjs.cloudflare.com`은 프로젝트 허용 CDN 도메인 (`infra_guide.md` 기준 확인 필요)
- `collect_home.js`는 SheetJS를 읽기(파싱) 전용으로 사용 중이므로, `collect_notice.js`에서 ExcelJS를 쓰기(생성) 전용으로 사용해도 충돌 없음
- 단, ExcelJS는 SheetJS보다 용량이 크므로 `collect_notice.html`에서만 로드하고 `base.html`에는 추가하지 않음

#### ExcelJS를 채택하는 경우 스타일 구현 예시

```javascript
// ExcelJS를 사용하는 경우 makeNotice() 내 스타일 처리 부분
const ExcelJS = window.ExcelJS;  // CDN 로드 후 전역 노출
const wb = new ExcelJS.Workbook();
const ws = wb.addWorksheet("환수내역");

// A1 제목 셀
const titleCell = ws.getCell("A1");
titleCell.value = `${branch} ${name} ${YYYY}년 ${_pad2(MM)}월 기준 환수내역`;
titleCell.font = { bold: true, size: 14, name: "맑은 고딕" };

// 3행 헤더 스타일
const headerRow = ws.getRow(3);
headerRow.font = { bold: true };
headerRow.fill = {
  type: "pattern", pattern: "solid",
  fgColor: { argb: "FFF6F8FB" }  // 테이블 헤더 배경 (#f6f8fb)
};

// 헤더+데이터 테두리 (3행 ~ 마지막 데이터 행, A~N열)
const borderStyle = { style: "thin", color: { argb: "FFD2DDEC" } };
const border = { top: borderStyle, left: borderStyle, bottom: borderStyle, right: borderStyle };

for (let r = 3; r <= 3 + mergedRows.length; r++) {
  for (let c = 1; c <= 14; c++) {
    ws.getCell(r, c).border = border;
  }
}

// Blob 생성 (ExcelJS는 async)
const buffer = await wb.xlsx.writeBuffer();
const blob = new Blob([buffer], { type: "application/octet-stream" });
storeResult(blob, `환수내역_${name}_${YYYY}년${_pad2(MM)}월.xlsx`);
```

#### SheetJS만 유지하는 경우 (최소 구현)

ExcelJS CDN 추가가 불가한 경우, SheetJS에서 부분적으로 지원하는 스타일만 적용하고 테두리는 미지원 사항임을 UX 안내로 대체합니다.

```javascript
// SheetJS 0.18.x 스타일 수동 주입 (폰트 한정)
ws["A1"].s = {
  font: { bold: true, sz: 14 },
  alignment: { horizontal: "left" }
};
// ※ 테두리는 커뮤니티 버전 미지원 — 다운로드 후 Excel에서 직접 적용 필요
```
